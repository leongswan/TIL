# 2026-01-02
## 今日学んだこと（ネットワーク基礎）

---

## 1) ネットワークアーキテクチャ（LAN / WAN）
- **LAN（Local Area Network）**：建物・拠点内などの近距離ネットワーク
- **WAN（Wide Area Network）**：拠点間・都市間など広域ネットワーク（インターネットもWANの一種）

---

## 2) ネットワーク形態（トポロジ）
| 方式 | 特徴 | 例 |
|---|---|---|
| **バス型** | 1本のケーブルを共有。障害に弱い | 旧Ethernet |
| **リング型** | 輪状に接続。1箇所障害で影響大（対策あり） | Token Ring |
| **スター型** | 中心機器に集中。管理しやすい | 現在のLANの主流 |

---

## 3) パケットとは / パケット交換方式
- **パケット**：データを小さく分割した単位（宛先情報などのヘッダ付き）
- **パケット交換方式**：パケット単位で送受信し、ネットワークを効率よく共有できる  
  - 回線を占有しないため、多数の通信を同時に流しやすい

---

## 4) 共有媒体でのアクセス制御
### CSMA/CD方式（主に旧Ethernet）
- **Carrier Sense**：送信前に回線が空いているか確認  
- **Multiple Access**：複数端末が同じ回線を使う  
- **Collision Detection**：衝突検出 → 再送  
※ 現在はスイッチ化で衝突が起きにくく、実運用では登場機会が減っている

### トークンパッシング方式
- 「**トークン（送信権）**」を持つ端末だけ送信できる  
- 衝突が起きにくいが、管理が複雑

---

## 5) キャスト（通信の届け方）
| 種類 | 意味 |
|---|---|
| **ユニキャスト** | 1台 → 1台 |
| **ブロードキャスト** | 1台 → 同一ネットワーク内の全台 |
| **マルチキャスト** | 1台 → 特定グループ |
| **エニーキャスト** | 1台 → 複数候補のうち「最も近い1台」 |

---

## 6) プロトコルとは / なぜ必要か
- **プロトコル**：通信で守るべき「**ルールの取り決め**」
- なぜ必要？
  - メーカーやOSが違っても「同じルール」で通信できる
  - データ形式・順序・誤り検出・再送などを統一できる
  - 事故（誤解・データ破損・順序入れ替わり）を防げる

---

## 7) TCP/IP と IPアドレス（IPv4 / IPv6）
### TCP/IPプロトコル群
- インターネットの標準的な通信方式の集合
- 代表例：**HTTP / FTP / TCP / IP / Ethernet** など

### IPアドレス
- ネットワーク上の「住所」
- **IPv4**：32bit（例：`192.168.0.1`）
- **IPv6**：128bit（例：`2001:db8::1`）  
  - アドレス枯渇問題を解決し、より多くの端末を扱える

---

## 8) OSI基本参照モデルとは？
- ネットワーク通信を **7階層** に分けて整理した「概念モデル」
- トラブル切り分けや、機器・プロトコルの役割理解に使う

### 低レイヤー / 高レイヤー
- **低レイヤー（下位層：1〜3層）**：物理〜ネットワークの基礎
- **高レイヤー（上位層：4〜7層）**：アプリに近い制御・サービス

### OSI 7層（役割）
| 層 | 名称 | 役割（キーワード） | 例 |
|---|---|---|---|
| 7 | アプリケーション | サービス提供 | HTTP / FTP |
| 6 | プレゼンテーション | データ形式・暗号化 | 文字コード / SSLの概念 |
| 5 | セッション | 通信の開始・維持・終了 | セッション管理 |
| 4 | トランスポート | 信頼性確保・ポート | **TCP / UDP** |
| 3 | ネットワーク | 異なるNW間の経路選択 | **IP / ルータ** |
| 2 | データリンク | 同一NW内の転送・MAC | Ethernet / スイッチ |
| 1 | 物理 | ケーブル・信号 | LANケーブル |

---

## 9) カプセル化とは
- 上位層 → 下位層へ渡すときに、各層が**ヘッダ情報を付け足す**仕組み  
- 受信側は逆にヘッダを外して復元する（デカプセル化）

---

## 10) ネットワーク構成機器（役割とレイヤ）
| 機器 | レイヤ | 役割 |
|---|---|---|
| **リピータ** | L1 | 信号を増幅して延長する（信号を増幅する） |
| **ハブ** | L1 | 受け取った信号を全ポートに中継（ブロードキャスト的） |
| **ブリッジ / L2スイッチ** | L2 | MACアドレスで転送先を選ぶ（同一NW内通信法） |
| **ルータ / L3スイッチ** | L3 | 異なるネットワーク間で経路決定・中継（パケット伝送） |
| **ゲートウェイ** | L4〜L7 | **プロトコル変換**を行う（異種ネットワーク接続） |
| **L4〜L7スイッチ** | L4〜L7 | サーバ負荷分散など高レイヤーでのスイッチ |

### 物理デバイス例
- **NIC**：端末に搭載されるネットワーク接続部品（MACアドレスはここに割り当て）
- **LANケーブル**：コンピュータを結ぶ物理ケーブル
- **リピータ**：信号を増幅して距離を伸ばす

---

## 11) よく出るプロトコル例
- **HTTP**：Web
- **FTP**：ファイル転送
- **TCP / UDP**：トランスポート層（信頼性の有無など）
- **IP（IPv4/IPv6）**：ネットワーク層
- **Ethernet / PPP**：データリンク層（LANや回線接続）

---

## 12) OpenFlow と SDN
- **SDN（Software-Defined Networking）**：ネットワーク制御をソフトウェアで集中管理する考え方
- **OpenFlow**：SDNを実現するために、スイッチに転送ルールを指示するための技術（代表的な方式）

## 問題21：配列A→B（流れ図のaに入る操作）
✅ **答え（aに入る操作）：`B(j, 7 - i) ← A(i, j)`**

### ポイント
図2（配列A）→図3（配列B）は、**行と列を入れ替えるだけ（転置）ではなく**  
**8×8の配列を「90度右回転（時計回り）」**させた配置になっています。

### 対応関係（図）
```text
A(i, j) を 90度右回転  →  B(j, 7 - i)

  i(行)↓  j(列)→             i(行)↓  j(列)→
  Aの(i, j) の要素  ----->    Bの(j, 7-i) に入る
```

## 問題22：再帰関数 f(5)

✅ **答え：15**

与えられた定義：

- n <= 1 なら 1

- それ以外は n + f(n-1)
計算：

- f(5) = 5 + f(4)

 - = 5 + 4 + f(3)

 - = 5 + 4 + 3 + f(2)

 - = 5 + 4 + 3 + 2 + f(1)

 - = 5 + 4 + 3 + 2 + 1

   = **15**

## 問題23：命令実行回数（1GHz / 平均0.8クロックで1命令）

✅ **答え：125,000（万命令/秒）（= 12.5億命令/秒）**

- 1GHz = 1秒あたり 1,000,000,000 クロック

- 1命令あたり平均 0.8クロック（CPI=0.8）なので

命令/秒：

- 1,000,000,000 ÷ 0.8 = 1,250,000,000（命令/秒）

「何万命令」なので：

- 1,250,000,000 ÷ 10,000 = 125,000（万命令/秒）

## 問題24：割込みA/Bがあるとき、メイン処理が使えるCPU時間

✅ **答え：2.0ミリ秒**

条件：

- 割込みA：0.5ms（優先度 高）

- 割込みB：1.5ms（優先度 低）

- 多重割込みあり（B実行中でもAが来たらAが優先）

発生タイミング（図より）：

- B：0ms と 5ms

- A：1ms、2.5ms、3.5ms

タイムライン（0〜5ms）
```text
0.0 ├─ B開始 ───────────────┐
    │  (0.0〜1.0 実行)      │
1.0 ├─ A(0.5ms) ─ 1.5       │  ※AがBを割り込む
1.5 ├─ B残り(0.5ms) ─ 2.0 ─┘  → B終了
2.0 ├─ Main(0.5ms) ─ 2.5
2.5 ├─ A(0.5ms) ─ 3.0
3.0 ├─ Main(0.5ms) ─ 3.5
3.5 ├─ A(0.5ms) ─ 4.0
4.0 ├─ Main(1.0ms) ─ 5.0
5.0 └─（この時点で区間終了。B発生はあるが「5msまで」の計算なので含めない）
```
Mainの合計：

- 2.0〜2.5 = 0.5

- 3.0〜3.5 = 0.5

- 4.0〜5.0 = 1.0
  
合計 **2.0ms**

## 問題26：稼働率の差（各装置0.9）

✅ **答え：0.18**

- 少なくともどちらか動けばOK（並列）

  - 1 − (1−0.9)×(1−0.9) = 1 − 0.1×0.1 = 0.99

- 2台とも動作必須（直列）

  - 0.9×0.9 = 0.81

- 差：0.99 − 0.81 = 0.18

## 問題27：バッファがオーバフローしない条件

✅ **答え：L ≧ (S − R) × T（S > R のとき）**
```text

送信(毎秒S) → [ バッファ(容量L) ] → 受信(毎秒R)

S > R なので、毎秒 (S - R) だけバッファに「たまり」が増える
T秒連続で送る → 最大たまり量 = (S - R) × T
だから L はそれ以上必要

```

## 問題28：優先度プリエンプティブ（リアルタイムOS）

## 問題29：増分バックアップ（incremental）

✅ **答え：
「直前のバックアップ以降に更新されたファイルだけをバックアップし、更新情報（更新フラグ）をリセットする」**

整理：

- 増分：更新分だけ取る＋フラグをリセットする

- 差分：更新分だけ取る＋フラグをリセットしない

## 問題30：DRAMの特徴

✅ 答え：

- メモリセル構造が単純で高集積化でき、ビット単価を安くできる

- （補足）リフレッシュ（定期的な再書込み）が必要なのがDRAMの重要ポイント

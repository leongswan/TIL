## 1. AWSマネジメントコンソールのホーム
AWSマネジメントコンソールのホーム画面では、  
**最近アクセスしたサービス、AWS Health、コストと使用状況** などを確認できる。

### 画面で確認できたもの
- 最近アクセスしたサービス
  - EC2
  - IAM
  - CloudWatch
  - RDS
  - Route 53
  - VPC など
- AWS Health
- コストと使用状況
- 今月の最高コスト

### 学んだこと
- AWSコンソールのホームは、**AWS利用状況の入口**になる画面
- よく使うサービスへすぐ移動できる
- コストや異常の有無も最初に確認できる

---

## 2. サービス一覧メニュー
画面左上の「サービス」メニューから、  
AWSの各種サービス一覧やカテゴリを確認できる。

### 代表的なカテゴリ
- コンピューティング
- ストレージ
- データベース
- ネットワーキングとコンテンツ配信
- セキュリティ、ID、およびコンプライアンス
- アプリケーション統合
- 分析
- 機械学習 など

### 学んだこと
- AWSには非常に多くのサービスがあるため、**カテゴリから探す**のが基本
- 「最近アクセスしたサービス」も表示されるため、よく使うサービスへ素早く移動できる

---

## 3. Billing（請求・コスト管理）
Billing Management Console では、  
**アカウント設定、請求、支払い、コスト管理** に関する情報を確認できる。

### 確認できた内容
- アカウント設定
- 連絡先情報
- お支払い通貨の設定
- 代替の連絡先
- 請求情報とコスト管理ダッシュボード
- サービス別利用料金の内訳

### 学んだこと
- AWSでは、**技術的な設定だけでなく費用管理も重要**
- 月額コストやサービスごとの利用額を可視化できる
- 予算超過や無駄な利用を防ぐために、Billing画面の確認は大切

---

## 4. Service Quotas
Service Quotas では、  
**AWSサービスごとの利用上限（クォータ）** を確認・管理できる。

### 画面で見た内容
- サービスごとのクォータ一覧
  - EC2
  - DynamoDB
  - EBS
  - RDS
  - VPC
  - Lambda
  - KMS など
- 保留中のサービスクォータリクエスト
- 最近承認されたクォータリクエスト
- クォータリクエストテンプレート

### 学んだこと
- AWSは無制限に使えるわけではなく、**サービスごとに上限がある**
- 必要に応じてクォータの引き上げ申請ができる
- 大規模構成では、事前に上限を把握しておくことが重要

---

## 5. IAMのセキュリティ認証情報
IAM Management Console の「セキュリティ認証情報」では、  
AWSアカウントの認証やアクセス管理に関わる設定を確認できる。

### 確認できた項目
- パスワード
- 多要素認証（MFA）
- アクセスキー
- X.509証明書
- CloudFrontキー
- アカウントID

### 学んだこと
- AWSでは、**認証情報の管理がとても重要**
- 特に以下が重要だと感じた
  - パスワードの適切な管理
  - MFAの有効化
  - アクセスキーの慎重な取り扱い
- セキュリティ面では、IAM設定を軽視してはいけない

---

## 6. AWSサポートセンター
AWSサポートセンターでは、  
**問い合わせ、ケース管理、重要通知の確認** ができる。

### 確認できた内容
- サポートケースの確認
- ケースの作成
- Healthイベント
- Compute Optimizer チェック
- Trusted Advisor チェック
- よくある質問

### 学んだこと
- AWSで困ったときに、**サポートセンターが相談窓口になる**
- 障害・通知・最適化情報もここから確認できる
- 単に質問を送る場所ではなく、運用支援の入口でもある

---

## 7. サポートプラン
AWSには複数のサポートプランがある。

### 確認できたプラン
- ベーシック
- デベロッパー
- ビジネス
- エンタープライズ

### 学んだこと
- プランによって、受けられるサポート内容が異なる
- 主な違いは以下のような点
  - 技術サポートの範囲
  - 応答時間
  - Trusted Advisor の利用範囲
  - Health Dashboard の利用範囲
- 個人学習と本番運用では、必要なサポートレベルが違う

---

## 8. AWSナレッジセンター
AWSナレッジセンターは、  
**AWSに関するよくある質問やトラブル解決情報を探せるサイト**。

### 特徴
- 人気のサービス別に調べられる
- カテゴリごとに情報を探せる
- 困りごとの自己解決に役立つ

### 学んだこと
- すぐに問い合わせる前に、**ナレッジセンターで調べる**のが有効
- 公式のFAQや解決策がまとまっているので信頼しやすい

---

## 9. AWS Documentation
AWS Documentation は、  
**AWS公式ドキュメントやAPIリファレンスを確認できるサイト**。

### 確認できた内容
- Guides and API References
- Compute
- Containers
- Storage
- Database などのカテゴリ別ドキュメント

### 学んだこと
- AWSを正しく理解するには、**公式ドキュメントを読む力**が必要
- 実装や設定の正確な確認には、公式情報が最も重要
- 今後学習を進めるうえで必ず使う場所になる

---

## 10. 実際のAWS Webサイトを見て分かったこと
今回、実際の画面を見て特に重要だと思ったのは、  
AWSは単なるクラウド実行環境ではなく、**運用・課金・制限管理・認証・サポート・学習資料まで一体化した大きな管理プラットフォーム** だということ。

### 特に印象に残った点
- コンソールホームは全体状況を確認する入口
- Billingは費用管理に必須
- Service Quotasは上限管理に重要
- IAMはセキュリティの中心
- サポートセンター、ナレッジセンター、公式ドキュメントが学習と運用を支える

---

## 今日の重要キーワード

| 用語 | 意味 |
|---|---|
| AWSマネジメントコンソール | Web画面からAWSを操作する管理画面 |
| Billing | 請求やコスト、支払い情報を管理する機能 |
| Service Quotas | AWSサービスごとの利用上限を管理する機能 |
| IAM | 認証・認可・アクセス管理を行う仕組み |
| MFA | 多要素認証。セキュリティ強化のための認証方式 |
| AWSサポートセンター | 問い合わせやサポートケース管理を行う画面 |
| サポートプラン | AWSが提供するサポート内容のレベル区分 |
| AWSナレッジセンター | よくある質問や解決策を探せる公式情報サイト |
| AWS Documentation | 公式ドキュメント・APIリファレンスを確認できるサイト |

---

## 今日の学び
- AWSコンソールのホームは、よく使うサービスやコスト確認の入口になる
- サービス一覧メニューから、カテゴリ別にAWSサービスを探せる
- Billing画面では請求やコストを確認でき、運用面で非常に重要
- Service Quotasでは利用上限を確認し、必要なら引き上げ申請できる
- IAMではパスワード、MFA、アクセスキーなど認証情報を管理する
- サポートセンターでは問い合わせや通知確認ができる
- サポートプランによって受けられる支援内容が異なる
- ナレッジセンターやDocumentationは、学習とトラブル解決に欠かせない

### 3.3 AWSのコンピューティングサービスを特定する

#### 知識
- AWSコンピューティングサービス

#### スキル
- EC2インスタンスタイプの適切な使い分けを理解する
  - コンピューティング最適化
  - ストレージ最適化 など
- コンテナサービスの使い分けを理解する
  - Amazon ECS
  - Amazon EKS
- サーバーレスコンピューティングの使い分けを理解する
  - AWS Fargate
  - AWS Lambda
- オートスケーリングによる伸縮性を理解する
- ロードバランサーの目的を理解する

#### 用語解説
- **インスタンス**：仮想サーバーの単位。EC2で作成した1つの仮想サーバー
- **コンテナ**：コンテナ型の仮想サーバー
- **オートスケーリング**：自動的に仮想サーバー数を増減させる機能
- **ロードバランサー**：トラフィック処理を制御・分散する機能

#### ポイント
- EC2、コンテナ、サーバーレスは用途によって使い分ける
- 負荷変動に対応するにはオートスケーリングとロードバランサーが重要

### 3.4 AWSのデータベースサービスを特定する

#### 知識
- AWSデータベースサービス
- データベース移行

#### スキル
- EC2上でホストするDBとマネージドDBのどちらを使うべきか判断できる
- リレーショナルデータベースを特定できる
  - Amazon RDS
  - Amazon Aurora
- NoSQLデータベースを特定できる
  - Amazon DynamoDB
- メモリベースのデータベースを特定できる
- データベース移行ツールを特定できる
  - AWS Database Migration Service（AWS DMS）
  - AWS Schema Conversion Tool（AWS SCT）

#### 用語解説
- **リレーショナルデータベース**：データ間の関連性が定義されたDB
- **NoSQLデータベース**：データ間の関連性が厳密に定義されず、SQL前提ではないDB
- **メモリデータベース**：メモリを利用したDBで、一般に読み書きが高速

#### ポイント
- DBの種類は、構造化データ・柔軟性・速度要件によって選ぶ
- 移行時には DMS や SCT のような専用ツールが役立つ

### 3.6 AWSのストレージサービスを特定する

#### 知識
- AWSストレージサービス

#### スキル
- オブジェクトストレージの用途を特定できる
- Amazon S3 のストレージクラスの違いを理解する
- ブロックストレージを特定できる
  - Amazon EBS
  - インスタンスストア
- ファイルサービスを特定できる
  - Amazon EFS
  - Amazon FSx
- キャッシュ済みファイルシステムを特定できる
  - AWS Storage Gateway
- ライフサイクルポリシーのユースケースを理解する
- AWS Backup のユースケースを理解する

#### 用語解説
- **オブジェクトストレージ**：オブジェクト形式でデータを保存するストレージ
- **ブロックストレージ**：ブロック単位でデータを保存するストレージ
- **ファイルサービス**：ファイル形式でデータを保存するストレージ
- **キャッシュ**：一時的に保存されるデータ
- **データライフサイクル**：保存データを削除したり保存先を変更したりするスケジュール

#### ポイント
- 保存方法には、オブジェクト・ブロック・ファイルの違いがある
- S3のライフサイクル管理やAWS Backupは運用効率化に役立つ

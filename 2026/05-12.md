# 2026/5/12 TIL

## 学習テーマ

AWS EBSの設定（ハンズオン）

## 今日学んだこと

AWSマネジメントコンソールを使って、EBSボリュームの確認・変更・作成・アタッチ・スナップショット作成・スナップショット削除の流れを学んだ。

EBSはEC2にアタッチして利用するブロックストレージで、容量やボリュームタイプを変更したり、スナップショットとしてバックアップを取得したりできる。

---

## 作業の流れ

1. EC2インスタンスを確認する
2. EBSボリューム一覧を開く
3. 既存ボリュームの情報を確認する
4. ボリュームの変更画面を確認する
5. 新しいEBSボリュームを作成する
6. 作成したEBSボリュームをEC2にアタッチする
7. EBSボリュームからスナップショットを作成する
8. スナップショットからボリュームを作成する
9. 不要なスナップショットを削除する

---

## EBSボリューム一覧を確認する

EC2の左メニューから以下を開く。

```text
Elastic Block Store > ボリューム
```

ボリューム一覧では、以下の情報を確認できる。

| 項目 | 内容 |
|---|---|
| ボリュームID | EBSボリュームを識別するID |
| タイプ | gp2 / gp3 / io1 / io2 など |
| サイズ | ボリューム容量 |
| IOPS | 1秒あたりの入出力性能 |
| スループット | データ転送性能 |
| スナップショット | 作成元スナップショット |
| 作成日 | ボリュームが作成された日時 |

---

## 既存ボリュームの確認

既存のEC2に紐づいているEBSボリュームを確認した。

例：

| 項目 | 内容 |
|---|---|
| ボリュームタイプ | gp2 |
| サイズ | 8GiB |
| IOPS | 100 |
| ステータス | 使用中 |
| ボリュームの状態 | OK |

既存のルートボリュームは、EC2インスタンス作成時に自動的に作成されている。

---

## ボリュームの変更

EBSボリュームを選択し、以下の操作を確認した。

```text
アクション > ボリュームの変更
```

変更画面では、以下を変更できる。

| 項目 | 内容 |
|---|---|
| ボリュームタイプ | gp2 / gp3 / io1 / io2 など |
| サイズ | EBSの容量 |
| IOPS | 入出力性能 |
| スループット | データ転送性能 |

---

## ボリュームタイプの選択

ボリュームタイプには、主に以下の種類がある。

| ボリュームタイプ | 特徴 |
|---|---|
| gp2 | 汎用SSD。以前からよく使われるタイプ |
| gp3 | 汎用SSD。IOPSやスループットを個別に設定しやすい |
| io1 | プロビジョンドIOPS SSD |
| io2 | 高耐久・高性能なプロビジョンドIOPS SSD |
| マグネティック | 旧世代のボリュームタイプ |

学習用や一般的な用途では、`gp2` や `gp3` を使うことが多い。

---

## 新しいEBSボリュームを作成する

新しいEBSボリュームを作成するには、以下をクリックする。

```text
ボリュームの作成
```

作成画面では、以下を設定する。

| 項目 | 設定例 |
|---|---|
| ボリュームタイプ | gp3 |
| サイズ | 1GiB |
| IOPS | 3000 |
| スループット | 125MiB/s |
| アベイラビリティゾーン | ap-northeast-1a |
| スナップショットID | 指定なし |
| 暗号化 | 必要に応じて有効化 |

---

## gp3ボリュームの特徴

今回のハンズオンでは、`gp3` のEBSボリュームを作成した。

| 項目 | 内容 |
|---|---|
| ボリュームタイプ | gp3 |
| サイズ | 1GiB |
| IOPS | 3000 |
| スループット | 125MiB/s |

`gp3` は、容量とは別にIOPSやスループットを設定できるため、性能を調整しやすい。

---

## アベイラビリティゾーンに注意する

EBSボリュームは、作成したAZに紐づく。

そのため、EC2インスタンスにアタッチするには、EC2とEBSが同じAZにある必要がある。

例：

| EC2のAZ | EBSのAZ | アタッチ可否 |
|---|---|---|
| ap-northeast-1a | ap-northeast-1a | 可能 |
| ap-northeast-1a | ap-northeast-1c | 不可 |

EBSを作成するときは、アタッチしたいEC2と同じAZを選択することが重要。

---

## EBSボリュームをアタッチする

作成したEBSボリュームを選択し、以下の操作を行う。

```text
アクション > ボリュームのアタッチ
```

アタッチ画面では、以下を指定する。

| 項目 | 内容 |
|---|---|
| ボリュームID | アタッチするEBSボリューム |
| アベイラビリティゾーン | 対象EC2と同じAZ |
| インスタンス | 接続先のEC2インスタンス |
| デバイス名 | Linux上で認識されるデバイス名 |

---

## スナップショットを作成する

EBSボリュームを選択し、以下の操作を行う。

```text
アクション > スナップショットの作成
```

スナップショット作成画面では、説明を入力する。

例：

```text
udemy
```

その後、以下をクリックする。

```text
スナップショットの作成
```

---

## スナップショットとは

スナップショットは、EBSボリュームのバックアップ。

EBSのデータを保存しておき、あとから新しいEBSボリュームとして復元できる。

| 項目 | 内容 |
|---|---|
| 作成元 | EBSボリューム |
| 保存先 | AWS管理のS3領域 |
| 用途 | バックアップ、復元、複製 |
| 復元 | スナップショットから新しいEBSボリュームを作成できる |

---

## スナップショット一覧を確認する

EC2の左メニューから以下を開く。

```text
Elastic Block Store > スナップショット
```

スナップショット一覧では、以下を確認できる。

| 項目 | 内容 |
|---|---|
| スナップショットID | スナップショットを識別するID |
| サイズ | バックアップ元ボリュームのサイズ |
| 説明 | 作成時に入力した説明 |
| ステータス | 完了済みなど |
| 開始時刻 | スナップショット作成日時 |

---

## スナップショットからボリュームを作成する

スナップショットを選択し、以下の操作を行う。

```text
アクション > スナップショットからボリュームを作成
```

ボリューム作成画面では、スナップショットIDが指定された状態で表示される。

確認する項目は以下。

| 項目 | 内容 |
|---|---|
| スナップショットID | 復元元のスナップショット |
| ボリュームタイプ | gp2 / gp3 など |
| サイズ | 復元するボリューム容量 |
| IOPS | 入出力性能 |
| アベイラビリティゾーン | 使用したいEC2と同じAZを選ぶ |
| 暗号化 | 必要に応じて有効化 |

---

## スナップショットを削除する

不要になったスナップショットは、以下の操作で削除できる。

```text
アクション > スナップショットの削除
```

削除確認画面が表示されるため、対象のスナップショットIDを確認して削除する。

削除後、スナップショット一覧から対象のスナップショットが消えていれば完了。

---

## 今日のポイント

- EBSボリュームはEC2にアタッチして使うストレージ。
- EBSボリュームはEC2と同じAZに作成する必要がある。
- ボリューム一覧から、サイズ・タイプ・IOPS・スループットを確認できる。
- `ボリュームの変更` から、ボリュームタイプやサイズを変更できる。
- `gp3` はIOPSやスループットを個別に設定しやすい。
- 新しいEBSボリュームは `ボリュームの作成` から作成する。
- 作成したEBSは `ボリュームのアタッチ` でEC2に接続できる。
- スナップショットを作成すると、EBSのバックアップを取得できる。
- スナップショットから新しいEBSボリュームを作成できる。
- 不要なスナップショットは削除して管理する。

---

## 自分用メモ

EBSの設定ハンズオンで覚える流れは以下。

1. EC2画面を開く
2. 左メニューから `ボリューム` を開く
3. 既存のEBSボリュームを確認する
4. `アクション > ボリュームの変更` で変更画面を確認する
5. `ボリュームの作成` で新しいEBSを作成する
6. ボリュームタイプ・サイズ・IOPS・AZを設定する
7. 作成したEBSをEC2にアタッチする
8. 必要に応じてスナップショットを作成する
9. スナップショットからEBSを復元する
10. 不要なスナップショットを削除する

EBSを操作するときは、特に **AZがEC2と同じかどうか** を確認することが重要。

# 2026/04/04 TIL

## 今日のテーマ
オンプレミスとAWSクラウドの良し悪し

---

## 概要
今日は、**オンプレミス環境とAWSクラウドのメリット・デメリット**を学んだ。  
特に重要だったのは、クラウドは**迅速かつ柔軟にリソースを使える**一方で、  
**ベンダーロックイン**という大きなデメリットがあること。

また、オンプレミスとクラウドは完全な二択ではなく、  
**ハイブリッド構成**や **AWS Outposts** のような仕組みによって、  
両方の良さを組み合わせることもできると理解した。

---

## 1. クラウドの主なメリット
クラウドの大きな利点は、**必要なリソースを必要な時に、すばやく柔軟に使えること**。

### ポイント
- オンデマンドでリソースを利用できる
- 物理機器の調達や設置を待たずに使い始められる
- 利用状況に応じて柔軟に拡張・縮小しやすい
- 自前でデータセンターを構築するより、高いセキュリティ水準を期待しやすい
- オンプレミス環境と組み合わせた**ハイブリッド構成**も可能

### 学んだこと
クラウドの強みは、**スピード**と**柔軟性**にある。  
インフラを自前で用意するよりも、素早くシステムを立ち上げやすい。

---

## 2. クラウドの主なデメリット
クラウドには便利さがある一方で、注意すべき弱点もある。

### 主なデメリット
- クラウド業者のサービスに依存してしまう **ベンダーロックイン**
- データの保存先や社内保管義務などのルールによっては、クラウドを使いにくい場合がある

### 学んだこと
クラウドは万能ではなく、  
**法規制・社内ルール・システム依存性**を考慮して使う必要がある。

---

## 3. ベンダーロックインとは
ベンダーロックインとは、  
**特定のクラウド事業者のサービスや仕組みに強く依存してしまい、他の環境へ移行しにくくなること**。

---

## 4. オンプレミス環境での移行のしやすさ
オンプレミスでシステムを構築している場合、  
サーバー変更やソフトウェア変更は**比較的容易**に行いやすい。

### 例
- 現行サーバーから新サーバーへ移行する
- Apache や Linux などのソフトウェアを入れ替える

### 学んだこと
オンプレミスでは、自社で環境を管理しているため、  
構成変更や移行の自由度が比較的高い。

---

## 5. AWSでも移行しやすいもの
AWSのサービスでも、**オンプレミスと類似した製品**は比較的移行しやすい。

### 例
- EC2 インスタンス

### ポイント
- EC2 は「仮想サーバー」に近い考え方なので、他環境への移行を考えやすい
- 標準的なサーバー利用に近い構成は、比較的ポータブル

### 学んだこと
AWSを使っていても、  
**IaaS寄りの構成**は移行しやすい場合がある。

---

## 6. AWS独自サービスを使うほど移行は難しくなる
AWS独自のサーバレスやマネージドサービスを深く組み込むと、  
他環境への移行が難しくなる。

### 例
- AWS Lambda
- Amazon SQS
- AWS API Gateway

### ポイント
- AWS独自の仕組みに合わせて開発すると、他クラウドや別ソフトウェアでそのまま再現しにくい
- AWSに深く依存した設計ほど、AWS外への移行が困難になる

### 学んだこと
便利なサービスほど依存度が高くなりやすいため、  
**開発効率と移行のしやすさのバランス**を考えることが大切。

---

## 7. ハイブリッド構成という考え方
オンプレミスとクラウドは対立関係ではなく、  
**組み合わせて使う**こともできる。

### ポイント
- オンプレミス環境とのハイブリッド構成が可能
- 一部をAWS、一部をオンプレミスに残す使い方もできる
- ルール上すべてをクラウド化できない場合の現実的な選択肢になる

### 学んだこと
要件に応じて、  
**全部クラウド**ではなく**必要なところだけクラウド**という考え方も重要。

---

## 8. AWS Outposts
AWS Outposts は、  
**オンプレミス環境にAWSサービスを展開できる仕組み**。

### できること
- 自社データセンターやオンプレミス環境にAWSサービスをセットアップできる
- AWSの運用感を保ちながら、オンプレ環境で利用できる

### 例
- IAM
- EC2
- RDS
- CloudWatch

### メリット
- 社内保管が必要なデータを扱いやすい
- オンプレミス要件とAWSの利便性を両立しやすい
- クラウド利用が難しい場面の回避策になりうる

### 学んだこと
クラウドが使いにくい制約があっても、  
**Outpostsのような仕組みで補える場合がある**。

---

## 9. オンプレミスとAWSクラウドの比較

| 観点 | オンプレミス | AWSクラウド |
|---|---|---|
| 導入スピード | 遅くなりやすい | 速い |
| 柔軟性 | 機器調達に左右されやすい | 高い |
| 移行のしやすさ | 比較的高い | サービス依存が強いと難しい |
| セキュリティ | 自社で全部対応 | 高い水準を活用しやすい |
| ルール対応 | 社内保管しやすい | 保存場所ルールに注意が必要 |
| 拡張性 | 物理制約がある | 拡張しやすい |

---

## 10. 今日の重要キーワード

| 用語 | 意味 |
|---|---|
| オンプレミス | 自社内でサーバーや機器を保有・運用する形態 |
| クラウド | インターネット経由でインフラやサービスを利用する形態 |
| ベンダーロックイン | 特定ベンダーのサービス依存により移行しにくくなること |
| ハイブリッド構成 | オンプレミスとクラウドを組み合わせる構成 |
| AWS Outposts | オンプレミス環境でAWSサービスを利用できる仕組み |
| EC2 | AWSの仮想サーバーサービス |
| Lambda | AWSのサーバレス実行サービス |
| SQS | AWSのメッセージキューサービス |
| API Gateway | API公開・管理のためのAWSサービス |

---

## 今日の学び
- クラウドの最大のメリットは、**迅速かつ柔軟にリソースを使えること**
- クラウドの大きなデメリットは、**ベンダーロックイン**
- EC2のような標準的なサーバー型サービスは比較的移行しやすい
- Lambda や API Gateway のようなAWS独自機能に深く依存すると移行が難しくなる
- 社内ルールやデータ保管要件によっては、クラウドを使いにくい場合がある
- その場合でも、**ハイブリッド構成**や **AWS Outposts** が回避策になりうる

---

## 一言まとめ
オンプレミスは**移行のしやすさや社内管理のしやすさ**があり、  
AWSクラウドは**スピード・柔軟性・高い利便性**が強み。  
重要なのは、要件に応じて **クラウド依存の深さを考えながら最適な構成を選ぶこと**。

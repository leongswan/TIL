# TIL（2026/2/26）C言語：関数（宣言 / 引数 / 戻り値）

## 1. 関数とは
- **処理をひとまとめにしたもの**（部品化）
- 同じ処理を何度も使える／見通しが良くなる／修正が楽になる

---

## 2. 関数の基本要素
- **戻り値の型**：関数が返す値のデータ型（例：`int`）
- **関数名**：呼び出すときの名前（例：`sum`）
- **引数（ひきすう）**：関数に渡す入力（例：`int a, int b`）
- **戻り値（return）**：関数の結果として返す値（例：`return a + b;`）

流れ（イメージ）：
```text
引数（入力） → 関数（処理） → 戻り値（出力）
```

---

## 3. 関数の宣言（プロトタイプ宣言）

- `main()` より後に関数定義を書く場合、先に**宣言**が必要
- 形だけ先に書いて「こういう関数がありますよ」とコンパイラに伝える

例：
```c
int sum(int, int);  // 引数の型だけでもOK（プロトタイプ宣言）
```

---

## 4. 戻り値がある関数の例（sum）
### 全体コード（宣言 → main → 定義）
```c
#include <stdio.h>

int sum(int, int);  // 宣言（プロトタイプ）

int main(void){
    int ans;
    ans = sum(2, 3);        // 呼び出し（実引数）
    printf("%d\n", ans);    // 5
    return 0;
}

int sum(int a, int b){      // 定義（仮引数）
    int ans;
    ans = a + b;
    return ans;             // return a + b; でもOK
}
```
### 用語の対応

- `sum(2, 3)` の**2, 3 → 実引数**
- `int a, int b` の**a, b → 仮引数**

---

## 5. 戻り値がない関数（void）の例

- `void` は「戻り値なし」
- `return;` は書いてもいいが、省略もできる

```c
#include <stdio.h>

void hello(void){
    printf("Hello\n");
}

int main(void){
    hello();
    return 0;
}
```

## 今日のまとめ

- 関数は「処理を部品化」して再利用しやすくする
- `戻り値の型 関数名(引数)` が基本形
- 関数を `main()` より後に書くなら **プロトタイプ宣言** が必要
- `void` は「戻り値なし」の意味

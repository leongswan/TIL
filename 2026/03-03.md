# TIL（2026/3/2）C言語：構造体ポインタ / 構造体配列 / ポインタ配列

## 1) 構造体（struct）とは
- 複数の型のデータを1つにまとめて扱うための仕組み（データのまとまり）
- 例：動物の **id / 名前 / 敵（別の動物への参照）** を1つの型として表現できる

```c
struct animal{
    int id;
    char name[100];
    struct animal *enemy;  // 同じstruct型へのポインタ（自己参照）
};
```

---

## 2) 構造体配列（structの配列）

- `struct animal animals[3];` のようにすると、animal型の要素が3個並ぶ配列になる
- 各要素には `{id, "name", enemy}` の形で初期値を入れられる
```c
struct animal animals[3] = {
    {1, "Buffalo", NULL},
    {2, "Dolphin", &animals[2]}, // enemyはGoatを指す
    {3, "Goat",    &animals[1]}  // enemyはDolphinを指す
};
```
### ポイント：`&animals[2]` の意味

- `animals[2]` は「3番目の要素（Goat）」
- `&animals[2]` は「その要素のアドレス（ポインタ）」
- つまり enemy に「別の要素を指す参照」を持たせられる

---

## 3) 要素アクセス：`.` と `->` の違い
### `.`（ドット演算子）

- **構造体そのもの**（実体）からメンバにアクセスするときに使う
```c
printf("%s\n", animals[1].name);   // 2番目の要素（Dolphin）のname
```
※あなたの例では `printf("%d\n",animals[1].name);` になっていますが、
`name` は文字列（char配列）なので `%s` を使うのが正しいです。

### `->`（アロー演算子）

- **構造体へのポインタ**からメンバにアクセスするときに使う（`(*ptr).name` と同じ意味）
```c
struct animal *animals_ptr = &animals[1];
printf("%s\n", animals_ptr->name);
```

---

## 4) 構造体ポインタで配列を順に処理する

- 配列の各要素のアドレスをポインタに入れて、`->` でアクセスする
```c
struct animal *animals_ptr;

for (int i = 0; i < 3; i++){
    animals_ptr = &animals[i];
    printf("%s\n", animals_ptr->name);
}
```

---

## 5) ポインタ配列（pointer array）とは

- 「ポインタの配列」＝アドレスを並べる配列
- 例：animalを指すポインタを並べる（順番を自由に変えたり、一部だけ選んだりできる）
```c
struct animal *p[3];
p[0] = &animals[2]; // Goat
p[1] = &animals[0]; // Buffalo
p[2] = &animals[1]; // Dolphin

for (int i = 0; i < 3; i++){
    printf("%s\n", p[i]->name);
}
```

---

## 今日のまとめ

- **構造体配列**：`struct animal animals[3]` で複数の要素をまとめて管理できる
- **構造体ポインタ**：`struct animal *ptr` で要素のアドレスを扱い、`->` でアクセスする
- **`.` と `->`**
   - `animals[i].name`：実体からアクセス（.）
   - `ptr->name`：ポインタからアクセス（->）
- **ポインタ配列**：`struct animal *p[]` で「参照の並び」を作れる（順番入れ替え・選択がしやすい）

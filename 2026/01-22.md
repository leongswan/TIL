## 問題1：10進数 7÷32 を2進数で表す
✅ **答え：0.00111₂**

### 考え方（超ラクな見方）
- 32 = 2⁵ なので、**7 を 5ビット右に小数としてシフト**するイメージ
- 7 = 111₂  
  → 111 を 5桁ぶん右へ：**0.00111₂**

確認：
- 0.00111₂ = 1/8 + 1/16 + 1/32 = 0.125 + 0.0625 + 0.03125 = **0.21875**
- 7/32 = **0.21875** ✅

## 問題2：最上位がパリティビットの8ビット符号から下位7ビットを取り出す
✅ **答え：`AND 0x7F`（16進数7FとのAND）**

### 理由
- 0x7F = 0111 1111₂  
- 最上位(パリティビット)だけを 0 にして、下位7ビットを残せる

~~~text
  b7 b6 b5 b4 b3 b2 b1 b0   (b7がパリティ)
  x  x  x  x  x  x  x  x
& 0  1  1  1  1  1  1  1   (=0x7F)
-------------------------
  0  x  x  x  x  x  x  x   ←下位7ビットだけ残る
~~~

## 問題3：命題の真理値（P真、(¬P)∨Q 真、(¬Q)∨R 真）
✅ **答え：Q = 真、R = 真**

### 手順
- P が真 → ¬P は偽  
  (¬P)∨Q が真になるには **Qが真**でないと無理（偽∨Q = Q）
- Q が真 → ¬Q は偽  
  (¬Q)∨R が真になるには **Rが真**

## 問題5：2分探索木として適切なもの
✅ **答え：イ**

### 2分探索木（BST）の条件
- どのノードでも  
  **左部分木 < 自分 < 右部分木** を満たす（再帰的に）

### 図（選択肢イの形）
~~~text
        4
      /   \
     2     8
    / \   / \
   1   3 6   9
        / \
       5   7
~~~
- 左側：1,2,3 はすべて 4 より小さい ✅
- 右側：5,6,7,8,9 はすべて 4 より大きい ✅
- 8 の左が 6（<8）、右が 9（>8）なども全部OK ✅

## 問題6：再帰関数 f() 実行後の B の状態（A,B,C初期：[1,2,3]）
✅ **答え：B = [1,2,3,1,2,3]**

### 関数の動き（要点）
- A が空になるまで  
  **Aからpop → Cへpush** を繰り返す（再帰で深く潜る）
- 戻りながら  
  **Cからpop → Bへpush** をする

~~~text
初期：
A=[1,2,3]  B=[1,2,3]  C=[1,2,3]

(潜る) A→Cへ移す：3,2,1 の順でCに積まれる
C=[1,2,3,3,2,1]

(戻る) C→Bへ移す：1,2,3 の順でBに追加
B=[1,2,3,1,2,3]
~~~

## 問題7：引き算版ユークリッド互除法（A=876, B=204）比較回数
✅ **答え：11回**

### 追跡（L=A, S=B。比較＝「L:S」を判定する回数）
~~~text
#  L    S    判定    更新
1  876  204  L>S     L=672
2  672  204  L>S     L=468
3  468  204  L>S     L=264
4  264  204  L>S     L=60
5  60   204  L<S     S=144
6  60   144  L<S     S=84
7  60   84   L<S     S=24
8  60   24   L>S     L=36
9  36   24   L>S     L=12
10 12   24   L<S     S=12
11 12   12   L=S     終了
~~~

## 問題10：実効アクセス時間が短い順（A〜D）
✅ **答え：A → D → B → C**

### 使う式
- **実効アクセス時間 = キャッシュ時間×ヒット率 + 主記憶時間×(1-ヒット率)**  
  ※キャッシュなしはそのまま主記憶時間

### 計算
~~~text
A：キャッシュなし → 15ns
B：キャッシュなし → 30ns
C：20×0.60 + 70×0.40 = 12 + 28 = 40ns
D：10×0.90 + 80×0.10 =  9 +  8 = 17ns
~~~
短い順：**15(A) < 17(D) < 30(B) < 40(C)**

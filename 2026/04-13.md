AWSでサーバーを構築してみる（ハンズオン）①

## 概要
今日は、AWSマネジメントコンソールを使って **EC2インスタンスを実際に作成し、サーバーを立ち上げる流れ** を学んだ。  
オンプレミスのサーバー調達と比べて、AWSでは必要な機能をオンデマンドで素早く利用できることを、実際の画面操作を通して確認した。

---

## 1. AWSの仕組み
AWSでは、インフラやアプリ開発に必要な機能が **パーツサービス** として提供されている。

### 例
- オンプレミスのサーバー → **EC2**
- オンプレミスのストレージ → **S3**

### 学んだこと
オンプレミスではサーバーやストレージを用意するのに時間もコストもかかるが、  
AWSでは必要なときに必要な分だけ、すぐに使い始められる。

---

## 2. AWSでサーバーを立ち上げるメリット
講義では、オンプレミスとAWSの違いを比較していた。

### オンプレミス
- 準備に時間がかかる
- コストがかかる

### AWS
- 数分で立ち上げ可能
- 無料利用枠を使って試せる場合がある

### 学んだこと
AWSの強みは、**スピード感を持ってサーバーを構築できること** にある。

---

## 3. AWSマネジメントコンソールからEC2を開く
まず AWS マネジメントコンソールに入り、検索バーで `EC2` を検索してサービスを開く。

### 流れ
1. AWSマネジメントコンソールにログイン
2. 検索バーで `EC2` を入力
3. EC2 ダッシュボードを開く
4. 「インスタンスを起動」を選ぶ

### 学んだこと
AWSはサービス数が多いため、  
**検索バーから必要なサービスへ直接移動する操作が便利**。

---

## 4. EC2インスタンス作成時の主な設定
インスタンス起動画面では、サーバー作成に必要な各項目を設定する。

### 今回の主な設定項目
- **名前とタグ**
  - 例: `udemy` などの名前を設定
- **AMI（OSイメージ）**
  - 講座では Amazon Linux 系AMI を選択していた
  - レクチャー案内では Amazon Linux 2023 の利用指定あり
- **インスタンスタイプ**
  - `t2.micro` を使用
- **キーペア**
  - 新規作成してダウンロード
- **ネットワーク設定**
  - セキュリティグループを作成
- **ストレージ**
  - ルートボリューム `8 GiB gp2`

### 学んだこと
サーバーは「起動ボタンを押すだけ」ではなく、  
**OS、サイズ、鍵、通信設定、ディスク容量をセットで決める必要がある**。

---

## 5. キーペアの作成
インスタンスへ安全に接続するために、キーペアを作成する。

### ポイント
- キーペア名を入力して作成
- 形式は `.pem` などを選ぶ
- 作成すると秘密鍵ファイルがダウンロードされる
- この秘密鍵は後でSSH接続に使う

### 学んだこと
キーペアはサーバー接続に必要な重要情報なので、  
**作成後は安全に保管することが必須**。

---

## 6. セキュリティグループの設定
セキュリティグループは、EC2への通信を制御するファイアウォールの役割を持つ。

### 今回の設定例
- **SSH を許可**
- **HTTP を許可**
- **HTTPS を許可**

### 補足
- SSH はリモートログイン用
- HTTP / HTTPS はWebアクセス用

### 学んだこと
サーバーを作るだけでなく、  
**どの通信を許可するかを明示的に設定しないと利用できない**。

---

## 7. ストレージ設定
インスタンスのルートボリュームとしてストレージ容量を設定する。

### 今回の設定
- `8 GiB`
- `gp2`

### 学んだこと
最初は最小構成で始めてもよいが、  
サーバー用途に応じて後からストレージ設計を考える必要がある。

---

## 8. インスタンスの起動
必要な設定が完了したら、「インスタンスを起動」を実行する。

### 起動後の流れ
- 起動中（Pending）
- 実行中（Running）
- 成功画面の表示
- 「View all instances」で一覧へ移動

### 学んだこと
EC2はすぐに使えるように見えても、  
**状態が Pending から Running に変わるまで少し待つ必要がある**。

---

## 9. 起動後に確認する情報
インスタンス一覧画面では、起動したサーバーの詳細が確認できる。

### 確認できる主な情報
- インスタンス名
- インスタンスID
- インスタンスの状態（実行中など）
- インスタンスタイプ
- アベイラビリティゾーン
- パブリック IPv4 アドレス
- パブリック IPv4 DNS
- プライベートIPアドレス

### 学んだこと
特に重要なのは、
- **状態が Running になっていること**
- **接続に使うIPアドレスやDNS名を確認すること**

## 今日の重要キーワード

| 用語 | 意味 |
|---|---|
| EC2 | AWS上で仮想サーバーを作成できるサービス |
| S3 | AWSのオブジェクトストレージサービス |
| AWSマネジメントコンソール | AWS各サービスをGUIで操作する管理画面 |
| AMI | インスタンス起動時に使うOSイメージ |
| インスタンスタイプ | サーバー性能のサイズ指定 |
| t2.micro | 小さい構成のインスタンスタイプ |
| キーペア | インスタンス接続に使う公開鍵・秘密鍵の組 |
| セキュリティグループ | EC2への通信を制御する仮想ファイアウォール |
| Pending | インスタンス起動中の状態 |
| Running | インスタンス実行中の状態 |
| パブリックIPv4 | インターネット経由で接続するときに使うIPアドレス |

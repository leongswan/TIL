# TIL（2026/3/7）C言語：`sizeof` 演算子（配列サイズ・要素数の求め方）

## 1. `sizeof` 演算子とは
- `sizeof` は、**型や変数がメモリ上で占めるバイト数**を返す演算子。
- 返り値の型は `size_t`（環境依存だが符号なし整数）なので、表示するときは本来 `"%zu"` を使うのが安全。

---

## 2. 2次元配列での `sizeof` の考え方
対象配列：
```c
int pat[3][2] = {
    {1,2},
    {1,2},
    {1,2}
};
```
### (1) 配列全体のサイズ（総バイト数）

- `sizeof pat`
→ **配列 `pat` 全体**が占めるバイト数
→ `3行 × 2列 × sizeof(int)` になる
```c
size_t a = sizeof pat;
```
### (2) 行数（第一次元の要素数）

- `sizeof pat / sizeof pat[0]`

  - `sizeof pat`：全体のバイト数
  - `sizeof pat[0]`：1行分（`int[2]`）のバイト数
  - なので割ると **行数（3）** が出る
```c
size_t b = sizeof pat / sizeof pat[0];
```
### (3) 全要素数（行×列）

- `sizeof pat / sizeof pat[0][0]`
  - `sizeof pat[0][0]` は `int` 1個のバイト数
  - 全体 ÷ 1要素サイズ → **要素数（3×2=6)**
```c
size_t c = sizeof pat / sizeof pat[0][0];
```

---

## 3. サンプル（出力まで含めた安全な書き方）
```c
#include <stdio.h>

int main(void){
    int pat[3][2] = {
        {1,2},
        {1,2},
        {1,2}
    };

    size_t a = sizeof pat;                  // 総バイト数
    size_t b = sizeof pat / sizeof pat[0];  // 行数
    size_t c = sizeof pat / sizeof pat[0][0]; // 要素数

    printf("sizeof(pat): %zu\n", a);
    printf("rows: %zu\n", b);
    printf("elements: %zu\n", c);

    return 0;
}
```

---

## 4. 注意点（超重要）

- この`「sizeof 配列 / sizeof 要素」`のテクニックは、**配列が「配列のまま」存在するスコープでだけ有効**。
- 関数に渡すと配列は **ポインタに変換（decay）** されるため、関数内で `sizeof` をすると「ポインタのサイズ」になってしまう。

例（関数内では期待通りに要素数を取れない）：
```c
void f(int pat[][2]){
    printf("%zu\n", sizeof pat); // ここは配列全体ではなく「ポインタのサイズ」になりやすい
}
```

---

## 5. 今日の結論

- `sizeof` は **バイト数を返す**。
- 2次元配列なら
  - **総バイト数**：`sizeof pat`
  - **行数**：`sizeof pat / sizeof pat[0]`
  - **要素数**：`sizeof pat / sizeof pat[0][0]`
- ただし **関数引数にすると崩れる**ので、要素数が必要なら別途引数で渡すのが定石。

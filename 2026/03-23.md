## 学んだ内容

### 4.1 AWSの料金モデルを比較する

#### 知識
- コンピューティング購入オプション
  - オンデマンドインスタンス
  - リザーブドインスタンス
  - スポットインスタンス
  - Savings Plans
  - 専用ホスト
  - ハードウェア専有インスタンス
  - キャパシティー予約
- データ転送料金
- ストレージオプションと階層

#### スキル
- 各コンピューティング購入オプションを使うべきケースを比較・判断できる
- リザーブドインスタンスの柔軟性を説明できる
- AWS Organizations におけるリザーブドインスタンスの動作を説明できる
- 受信データ転送コストと送信データ転送コストを理解する
  - リージョン間転送
  - 同一リージョン内転送 など
- ストレージオプションやストレージ階層ごとの料金の違いを理解する

#### ポイント
- AWSは従量課金が基本だが、購入方法によってコスト最適化の方法が変わる
- 長期利用ならリザーブドインスタンスやSavings Plansが有利になりやすい
- 一時的で中断可能な処理にはスポットインスタンスが向いている
- データ転送やストレージ階層もコストに大きく影響する

### 4.2 請求、予算、およびコスト管理のためのリソースを理解する

#### 知識
- 請求サポートと情報
- AWSサービスの料金に関する情報
- AWS Organizations
- AWSコスト配分タグ

#### スキル
- 次のサービスの用途と機能を理解する
  - AWS Budgets
  - AWS Cost Explorer
  - AWS Billing Conductor
- AWS料金見積りツールの用途と機能を理解する
- AWS Organizations の一括請求
  - コンソリデーテッドビリング
- コスト配分の考え方を理解する
- コスト配分タグと請求レポートの関係を理解する
  - AWS Cost and Usage Report など

#### ポイント
- コスト管理では「使った後に確認する」だけでなく、事前見積りと予算管理が重要
- AWS Budgets は予算超過の監視に役立つ
- AWS Cost Explorer は利用料金の分析に役立つ
- コスト配分タグを使うと、部署・案件・環境ごとに費用を整理しやすい
- Organizations を使うと複数アカウントの請求をまとめて管理できる

## 重要用語まとめ

| 用語 | 意味 |
|---|---|
| オンデマンドインスタンス | 必要なときに使い、使った分だけ課金される購入方法 |
| リザーブドインスタンス | 長期利用を前提に割引を受けられる購入方法 |
| スポットインスタンス | 空きキャパシティを安価で利用できるが、中断の可能性がある購入方法 |
| Savings Plans | 一定利用を約束することで割引を受けられる料金モデル |
| コンソリデーテッドビリング | 複数AWSアカウントの請求を一括でまとめる仕組み |
| コスト配分タグ | リソースにタグを付けて費用を分類・集計しやすくする仕組み |
| AWS Budgets | 予算を設定して超過を監視するサービス |
| AWS Cost Explorer | AWS利用料金を可視化・分析するサービス |
| AWS Billing Conductor | 複数アカウントの請求表示や請求ルール管理を行うサービス |
| AWS Marketplace | AWS上で利用できる製品やソリューションを探して導入できるサービス |

## 今日の学び

- コスト管理には、見積り・予算・請求分析・タグ管理が必要
- AWS Organizations を使うと複数アカウントの請求をまとめて管理できる
- AWSには公式ドキュメント、re:Post、サポートプランなど多くの支援手段がある
- Trusted Advisor や AWS Health は、運用改善や障害把握に役立つ

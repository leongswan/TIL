## 今日学んだこと

EC2インスタンスに接続し、Webサーバーをインストールして、ブラウザからWebページを表示する流れを学んだ。

今回は、EC2上でApache HTTP Serverを起動し、`index.html` を作成して、パブリックIPアドレスから `hello world!!` を表示させた。

---

## 作業の流れ

1. EC2インスタンスへ接続する
2. rootユーザーに切り替える
3. パッケージをアップデートする
4. Apache HTTP Serverをインストールする
5. Web公開用ディレクトリへ移動する
6. `index.html` を作成する
7. Webサーバーを起動する
8. Webサーバーを自動起動設定にする
9. ブラウザでパブリックIPアドレスにアクセスする

---

## rootユーザーに切り替える

EC2にログインした後、管理者権限で作業するためにrootユーザーへ切り替えた。

```bash
sudo su
```

### コマンドの意味

| コマンド | 意味 |
|---|---|
| `sudo` | 管理者権限でコマンドを実行する |
| `su` | ユーザーを切り替える |
| `sudo su` | rootユーザーに切り替える |

rootユーザーになると、プロンプトが以下のように変わる。

```bash
[root@ip-xxx-xxx-xxx-xxx ec2-user]#
```

---

## パッケージをアップデートする

サーバー内のパッケージを最新状態に更新する。

```bash
yum update -y
```

### コマンドの意味

| 部分 | 意味 |
|---|---|
| `yum` | Amazon Linuxでパッケージを管理するコマンド |
| `update` | インストール済みパッケージを更新する |
| `-y` | 確認に自動でyesと答える |

アップデートが完了すると、以下のように表示される。

```bash
Complete!
```

---

## Apache HTTP Serverをインストールする

WebサーバーとしてApacheをインストールする。

```bash
yum install httpd
```

または、確認を省略する場合は以下のようにする。

```bash
yum install httpd -y
```

### コマンドの意味

| 部分 | 意味 |
|---|---|
| `yum install` | パッケージをインストールする |
| `httpd` | Apache HTTP Serverのパッケージ名 |

インストールが完了すると、以下のように表示される。

```bash
Installed:
  httpd.x86_64

Complete!
```

---

## Web公開用ディレクトリへ移動する

ApacheでWebページを公開するためのディレクトリへ移動する。

```bash
cd /var/www/html
```

### `/var/www/html` とは

`/var/www/html` は、Apacheで公開するHTMLファイルを置く場所。

このディレクトリに `index.html` を作成すると、ブラウザからアクセスしたときに表示される。

---

## ファイル一覧を確認する

```bash
ls
```

### コマンドの意味

| コマンド | 意味 |
|---|---|
| `ls` | 現在のディレクトリ内のファイルやフォルダを表示する |

最初は何も表示されなかったため、まだファイルが存在していないことを確認した。

---

## index.htmlを作成する

`nano` エディタを使って、`index.html` を作成する。

```bash
nano index.html
```

### コマンドの意味

| 部分 | 意味 |
|---|---|
| `nano` | ターミナル上で使えるテキストエディタ |
| `index.html` | Webページとして表示するHTMLファイル |

---

## HTMLを書く

`nano` が開いたら、以下のHTMLを入力した。

```html
<html><h1>hello world!!</h1></html>
```

### HTMLの意味

| 部分 | 意味 |
|---|---|
| `<html>` | HTML文書であることを表す |
| `<h1>` | 大きな見出しを表示するタグ |
| `hello world!!` | ブラウザに表示したい文字 |
| `</h1>` | h1タグの終了 |
| `</html>` | HTML文書の終了 |

---

## nanoで保存する

`nano` で編集した内容を保存する。

基本操作は以下。

| 操作 | 意味 |
|---|---|
| `Ctrl + O` | 保存する |
| `Enter` | ファイル名を確定する |
| `Ctrl + X` | nanoを終了する |

---

## index.htmlが作成されたか確認する

```bash
ls
```

作成後に `ls` を実行すると、以下のように表示された。

```bash
index.html
```

これで、`/var/www/html` に `index.html` が作成されたことを確認できた。

---

## Webサーバーを起動する

Apacheを起動する。

```bash
systemctl start httpd
```

### コマンドの意味

| 部分 | 意味 |
|---|---|
| `systemctl` | Linuxのサービスを管理するコマンド |
| `start` | サービスを起動する |
| `httpd` | Apache HTTP Server |

---

## Webサーバーを自動起動に設定する

EC2を再起動した後もApacheが自動で起動するように設定する。

```bash
systemctl enable httpd
```

実行すると、以下のようにシンボリックリンクが作成される。

```bash
Created symlink from /etc/systemd/system/multi-user.target.wants/httpd.service to /usr/lib/systemd/system/httpd.service.
```

### コマンドの意味

| 部分 | 意味 |
|---|---|
| `enable` | サービスを自動起動に設定する |
| `httpd` | Apache HTTP Server |

---

## ブラウザでWebサイトを確認する

EC2インスタンスのパブリックIPv4アドレスを確認し、ブラウザに入力する。

```text
http://パブリックIPv4アドレス
```

例：

```text
http://54.xxx.xxx.xxx
```

ブラウザに以下のように表示されれば成功。

```text
hello world!!
```

## 今日のポイント

- EC2上にWebサーバーを構築するには、Apache HTTP Serverである `httpd` をインストールする。
- Webページとして表示したいHTMLファイルは `/var/www/html` に配置する。
- トップページとして表示するファイル名は `index.html` にする。
- `nano index.html` でHTMLファイルを作成・編集できる。
- `systemctl start httpd` でWebサーバーを起動する。
- `systemctl enable httpd` でEC2再起動後も自動でApacheが起動するようにできる。
- ブラウザでEC2のパブリックIPv4アドレスにアクセスすると、作成したWebページを確認できる。

---

## 自分用メモ

EC2でWebサイトを表示させる基本手順は以下。

1. EC2へ接続する
2. `sudo su` でrootユーザーになる
3. `yum update -y` でパッケージを更新する
4. `yum install httpd -y` でApacheをインストールする
5. `/var/www/html` に移動する
6. `index.html` を作成する
7. HTMLを書く
8. `systemctl start httpd` でApacheを起動する
9. `systemctl enable httpd` で自動起動設定をする
10. ブラウザでパブリックIPアドレスにアクセスする

今回、ブラウザに `hello world!!` が表示されたので、EC2上でWebサーバーを構築してWebページを公開できた。

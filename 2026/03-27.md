## 学んだ内容

### 1. AWSクラウドの利点
グローバルに展開している企業がAWSクラウドを使う利点として、次の2つが正解だった。

#### 正解
- **1) AWSリージョンにアプリケーションを即時に展開できる**
- **3) エッジロケーションを利用してグローバルにアクセス提供やコンテンツ配信ができる**

#### 学んだこと
- AWSは世界中にリージョンを持っているため、各地域へ素早くサービス展開できる
- CloudFront などのエッジロケーションを使うことで、ユーザーに近い場所からコンテンツを配信できる
- AWSの利点は「売上が上がる」こと自体ではなく、**グローバル展開しやすい基盤があること**にある

---

### 2. AWSクラウドの設計原則
Amazon RDS でログ取得を有効化し、CloudWatch Logs でログ分析を行う戦略は、  
AWS Well-Architected フレームワークのどの柱に当てはまるかを学んだ。

#### 正解
- **3) オペレーショナルエクセレンス（運用上の優秀性）**

#### 学んだこと
- ログ取得やログ分析は、運用の可視化と改善に役立つ
- オペレーショナルエクセレンスは、
  - 運用状況を把握する
  - 問題を見つけやすくする
  - 継続的に運用を改善する  
  という考え方につながる
- CloudWatch Logs は監視や分析の重要なサービス

---

### 3. AWSへの移行サービス
オンプレミスの Linux サーバー上にあるデータベースサーバーを、  
AWS の Amazon EC2 に移行したい場合に使う適切なサービスを学んだ。

#### 正解
- **3) AWS Application Migration Service（AWS MGN）**

#### 学んだこと
- AWS MGN は、サーバー全体をAWSへ移行する用途に向いている
- AWS DMS はデータベースの**データ移行**に強いが、今回は Linux サーバー上の構成ごと移したいので MGN が適切
- 「データだけ移す」のか、「サーバーごと移す」のかで使うサービスが変わる

#### サービスの使い分け
- **AWS MGN**：サーバーやアプリケーション全体の移行
- **AWS DMS**：データベースのデータ移行
- **AWS Migration Hub**：移行状況の管理
- **AWS SMS**：旧来のサーバー移行サービス

---

### 4. AWSクラウドエコノミクス
コンピューティング負荷が変動するアプリケーションで、  
AWSが従来型データセンターより経済的である理由を学んだ。

#### 正解
- **3) Amazon EC2インスタンスは、必要に応じて起動できる**

#### 学んだこと
- AWSでは必要なときだけインスタンスを起動し、不要になれば停止・削除できる
- オンプレミスのようにピーク時に合わせて常に大きな設備を持つ必要がない
- これにより、変動するワークロードに対して無駄なコストを抑えやすい

#### ポイント
- AWSは**従量課金**と**伸縮性**が大きな特徴
- 負荷変動に合わせてリソース量を調整できるため、コスト最適化しやすい

## 今日の重要キーワード

| 用語 | 意味 |
|---|---|
| AWSリージョン | AWSのサービスを提供する地域単位 |
| エッジロケーション | ユーザーに近い場所でコンテンツ配信を行う拠点 |
| CloudFront | AWSのCDNサービス |
| CloudWatch Logs | ログの収集・監視・分析を行うサービス |
| オペレーショナルエクセレンス | 運用を継続的に改善する Well-Architected の柱 |
| AWS MGN | サーバーやアプリケーション全体をAWSへ移行するサービス |
| AWS DMS | データベースのデータ移行を行うサービス |
| EC2 | AWS上で仮想サーバーを動かすサービス |
| 従量課金 | 使った分だけ料金を支払う考え方 |

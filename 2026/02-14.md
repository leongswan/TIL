# TIL（2026/02/14）— Pythonのデータ型（リスト・タプル・辞書）と添字/辞書エラー

## 今日の学び（要点）
- **リスト（list）**
  - 複数データを関連付けて保存できる
  - **変更可能（ミュータブル）**
  - 基本的に「同じ種類のデータを並べて保持したい」ときに使う

- **タプル（tuple）**
  - 複数データを関連付けて保存できる
  - **変更できない（イミュータブル）**
  - 基本的に「異なる種類の変化しないデータ」をまとめて持ちたいときに使う

- **辞書（dict）**
  - **キー（key）と値（value）をペア**にして保持する
  - 値としてリストやタプルなどを持つこともできる
  - 「体系的にデータを管理したい」ときに使う（名前→点数、ID→ユーザーなど）

## 問題1：リストの添字（index）
### 問題
```python
name = ["田中","佐藤","山田"] 
print(name[ ? ])
```

### ポイント
- リストの添字は 0から始まる

  - name[0] = "田中"

  - name[1] = "佐藤"

  - name[2] = "山田"

### 答え
- 出力を "佐藤" にするには 1

## 問題2：辞書のエラー理由
### 問題（このままだとエラー）
```python
students = {1:"太郎"4:"次郎"2:"太郎"1:"史郎"}
print(students[4])
```
### エラーの理由（結論）
- 辞書の定義文法が誤っているから

  - dictでは要素（キー:値）の区切りに カンマ , が必要

  - 例：{1:"太郎", 4:"次郎"} のように書く

### 正しい書き方（修正版）
```python
students = {1:"太郎", 4:"次郎", 2:"太郎", 1:"史郎"}
print(students[4])
```
### 補足：キー重複について
- 上の修正版は キー 1 が重複している（1:"太郎" と 1:"史郎"）

- Pythonの辞書は 後から書いた方で上書きされる
   → students[1] は "史郎" になる

- ただし、**今回のエラー原因は「キー重複」ではなく「カンマがない文法エラー」**である

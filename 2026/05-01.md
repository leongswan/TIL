## 学習テーマ

AWSのMACによる操作②

## 今日学んだこと

MacのTerminalを使って、AWS EC2インスタンスへSSH接続する実際の流れを学んだ。

前回学んだ以下の操作を、実際のAWS管理画面とTerminalで確認した。

1. EC2インスタンスのパブリックIPv4アドレスを確認する
2. MacのTerminalを開く
3. 秘密鍵ファイルを `.ssh` フォルダへ移動する
4. 秘密鍵ファイルの権限を変更する
5. `ssh -i` コマンドでEC2へ接続する
6. 初回接続時の確認メッセージに `yes` と入力する

---

## EC2インスタンスの情報を確認する

AWSマネジメントコンソールで、EC2インスタンスの一覧画面を開き、接続したいインスタンスを選択する。

確認する主な情報は以下。

| 項目 | 内容 |
|---|---|
| インスタンスID | EC2インスタンスを識別するID |
| インスタンスの状態 | `実行中` になっていることを確認 |
| パブリックIPv4アドレス | SSH接続時に指定するIPアドレス |
| アベイラビリティゾーン | インスタンスが配置されているAZ |

SSH接続するには、EC2インスタンスが `実行中` である必要がある。

---

## MacのTerminalを開く

Macでは、Terminalを使ってEC2へSSH接続する。

Terminalを開くと、以下のような画面になる。

```bash
ユーザー名@localhost ~ %
```

ここから、EC2接続用のコマンドを入力していく。

---

## 秘密鍵ファイルを `.ssh` フォルダへ移動する

ダウンロードした `.pem` ファイルを、SSH用のフォルダへ移動する。

```bash
mv ~/Downloads/udemysample.pem ~/.ssh
```

### コマンドの意味

| 部分 | 意味 |
|---|---|
| `mv` | ファイルを移動するコマンド |
| `~/Downloads/udemysample.pem` | ダウンロードフォルダにある秘密鍵ファイル |
| `~/.ssh` | SSH関連のファイルを置くフォルダ |

`udemysample.pem` の部分は、自分の秘密鍵ファイル名に変更する。

---

## 秘密鍵ファイルの権限を変更する

SSH接続では、秘密鍵ファイルの権限が広すぎると接続できないことがある。

そのため、以下のコマンドで権限を `400` に変更する。

```bash
chmod 400 ~/.ssh/udemysample.pem
```

### コマンドの意味

| 部分 | 意味 |
|---|---|
| `chmod` | ファイルの権限を変更するコマンド |
| `400` | 所有者だけが読み取り可能な権限 |
| `~/.ssh/udemysample.pem` | 権限を変更する秘密鍵ファイル |

秘密鍵は他人に見られてはいけない重要なファイルなので、厳しい権限に設定する。

---

## EC2へSSH接続する

秘密鍵ファイルを指定して、EC2インスタンスへSSH接続する。

```bash
ssh -i ~/.ssh/udemysample.pem ec2-user@<パブリックIPv4アドレス>
```

### 例

```bash
ssh -i ~/.ssh/udemysample.pem ec2-user@xxx.xxx.xxx.xxx
```

### コマンドの意味

| 部分 | 意味 |
|---|---|
| `ssh` | SSH接続を行うコマンド |
| `-i` | 使用する秘密鍵ファイルを指定するオプション |
| `~/.ssh/udemysample.pem` | 使用する秘密鍵ファイル |
| `ec2-user` | EC2へログインするユーザー名 |
| `<パブリックIPv4アドレス>` | EC2インスタンスのパブリックIPアドレス |

Amazon Linux系のEC2では、ログインユーザー名として `ec2-user` を使う。

---

## 初回接続時の確認メッセージ

初めてEC2へSSH接続すると、以下のような確認メッセージが表示される。

```bash
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

この場合は、以下のように入力する。

```bash
yes
```

すると、接続先ホストが `known_hosts` に登録される。

```bash
Warning: Permanently added 'xxx.xxx.xxx.xxx' to the list of known hosts.
```

---

## 接続成功の確認

接続に成功すると、TerminalにAmazon Linuxの表示が出る。

```bash
Amazon Linux 2 AMI
```

その後、プロンプトが以下のように変わる。

```bash
[ec2-user@ip-xxx-xxx-xxx-xxx ~]$
```

この表示になれば、MacからEC2インスタンスへSSH接続できている。

---

## 今日のポイント

- EC2へSSH接続するには、パブリックIPv4アドレスを確認する。
- MacではTerminalを使ってSSH接続する。
- `.pem` ファイルは `.ssh` フォルダに移動して管理する。
- 秘密鍵ファイルは `chmod 400` で権限を制限する。
- EC2へ接続するときは `ssh -i` で秘密鍵を指定する。
- 初回接続時は確認メッセージに `yes` と入力する。
- 接続後に `[ec2-user@...]$` のような表示になれば、EC2へログインできている。

## よく使うコマンドまとめ

```bash
mv ~/Downloads/udemysample.pem ~/.ssh
chmod 400 ~/.ssh/udemysample.pem
ssh -i ~/.ssh/udemysample.pem ec2-user@<パブリックIPv4アドレス>
```

## 自分用メモ

MacからEC2へ接続する流れは以下。

1. AWS管理画面でEC2インスタンスを確認する
2. パブリックIPv4アドレスをコピーする
3. Terminalを開く
4. `.pem` ファイルを `.ssh` フォルダへ移動する
5. `chmod 400` で秘密鍵の権限を変更する
6. `ssh -i` コマンドでEC2へ接続する
7. 初回接続時は `yes` を入力する

秘密鍵ファイル名とパブリックIPv4アドレスは、自分の環境に合わせて変更すること。

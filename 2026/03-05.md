# TIL（2026/3/5）C言語：構造体・ポインタ・配列・再帰で「列車運行」をシミュレーション

## 1. 何をするプログラムか
- 鉄道の**区間（block）**をつないだ路線上で、**列車（train）**を進行させるシミュレーション。
- 各区間の入口には**信号機（signal）**があり、表示は `RED / GREEN`。
- 列車は「次区間に進める条件」を満たすと1区間進む。目的地（終着駅の出口区間）に到達した列車は取り除かれる。

---

## 2. 定数（信号の色）
- 信号は「赤=進入不可」「緑=進入可」みたいに扱う。
- Cのマクロ定義は **`#define NAME value`** が基本。

✅ 正しい書き方（例）
```c
#define RED   0
#define GREEN 1
```
---

## 3. 構造体設計（重要）
### `train_info`（列車）

- `number`：列車番号
- `dest`：終点（終着駅を出口とする区間）へのポインタ

### `block_info`（区間）

- `train`：区間内にいる列車へのポインタ（いなければ `NULL`）

- `next[2]`：次に接続する区間へのポインタ配列（分岐対応、最大2本）

- `signal`：その区間の信号表示（`RED` or `GREEN`）

ポイント：`next` が配列なので「1つだけ次がある」「2つ分岐がある」どちらも表現できる。

---

## 4. `set_signals()`：区間の状態から信号を更新

- **区間に列車がいるなら RED**
- **区間に列車がいないなら GREEN**
```c
void set_signals(struct block_info blocks[], int nblocks){
    for(int i=0; i<nblocks; i++){
        struct block_info *block = &blocks[i];
        if(block->train == NULL){
            block->signal = GREEN;
        } else {
            block->signal = RED;
        }
    }
}
```

---

## 5. `proceed()`：進める列車を判定して進行させる（本体）

やっていること（流れ）：

1.**後ろ（終点側）から**区間を走査する（`i=nblocks-1` → `0`）
- こうすると「同じステップで列車が2回動く」事故を起こしにくい

2.区間に列車がいなければ `continue`

3.その列車が **目的区間（dest）にいるなら**取り除く（`block->train = NULL`）

4.次区間候補 `next[0], next[1]` を順に見る

- 次区間が `NULL` ならスキップ

- 次区間の信号が `RED` ならスキップ（進入不可）

- `find_block(next, dest)==1` なら「目的地へ行けるルート上」→ 進行させる

  - `next->train = block->train`

  - `next->signal = RED`（すぐ赤にして他列車の進入を防ぐ）

  - `block->train = NULL`

  - `break`（1回進めたら分岐探索を終える）

---

## 6. `find_block()`：次区間から目的地へ到達できるか（再帰）

- 引数 `block` から `dest` に**到達可能か**を判定する関数。
- これは **再帰DFS（深さ優先探索）** の考え方。
```c
int find_block(struct block_info *block, struct block_info *dest){
    if(block == dest) return 1;   // もう目的地ならOK

    for(int i=0; i<2; i++){
        if(block->next[i] == NULL) continue;
        if(find_block(block->next[i], dest) == 1) return 1;
    }
    return 0;
}
```

---

## 7. `start()`：1サイクル（信号更新 → 進行 → 信号更新）を繰り返す

- 最初に `set_signals()`
- その後、`proceed()` と `set_signals()` を複数回回してシミュレーションを進める
```c
void start(struct block_info blocks[], int nblocks){
    set_signals(blocks, nblocks);
    for(int i=0; i<4; i++){
        proceed(blocks, nblocks);
        set_signals(blocks, nblocks);
    }
}
```

---

## 8. 学び（今日の重要ポイント）

- **構造体 + ポインタ**で「実世界のモノ（列車・区間）」をモデル化できる
- `next[2]` のような **ポインタ配列**で分岐路線を表現できる
- `->`（ポインタアクセス）と `.`（実体アクセス）の使い分けが必須
- `find_block()` のように **再帰**で「経路上かどうか」を判定できる
- **更新順序（後ろから処理 / 進行直後に信号を赤）**が衝突防止に効く

---

## 9. 注意点（つまずきやすい）

- `#define RED(0)` のような書き方は基本NG → `#define RED 0` の形にする
- `struct train_info` が `struct block_info *dest` を持つなら、状況によっては **前方宣言**が必要
- `system("pause")` はWindows依存なので、環境によっては動かない（移植性に注意）

  出口でほかの区間と接続する区間にいる列車を進行させた場合、進行した区間の信号機の表示を赤にする
  
  blocks

  void proceed(struct block_info blocks[],int nblocks){ 
  　　　int i,j; 
     　struct block_info *block;

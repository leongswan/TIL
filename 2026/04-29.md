AWSのMACによる操作①

## 今日学んだこと

MacのTerminalを使って、AWS EC2インスタンスへSSH接続するための基本操作を学んだ。

主に以下の3つの流れを確認した。

1. ダウンロードした秘密鍵ファイルを `.ssh` フォルダへ移動する
2. 秘密鍵ファイルの権限を変更する
3. 秘密鍵を指定してEC2へSSH接続する

---

## Mac Terminalでの接続準備

### 1. 秘密鍵ファイルを `.ssh` フォルダへ移動する

```bash
mv ~/Downloads/udemysample.pem ~/.ssh/
```

### コマンドの意味

| 部分 | 意味 |
|---|---|
| `mv` | ファイルを移動するコマンド |
| `~/Downloads/udemysample.pem` | ダウンロードフォルダにある秘密鍵ファイル |
| `~/.ssh/` | SSH接続で使う設定・鍵ファイルを置くフォルダ |

`udemysample.pem` の部分は、自分がダウンロードした `.pem` ファイル名に変更する。

---

## 秘密鍵ファイルの権限を変更する

### 2. 秘密鍵の権限を `400` に変更する

```bash
chmod 400 ~/.ssh/udemysample.pem
```

### コマンドの意味

| 部分 | 意味 |
|---|---|
| `chmod` | ファイルの権限を変更するコマンド |
| `400` | 所有者だけが読み取りできる権限 |
| `~/.ssh/udemysample.pem` | 権限を変更する秘密鍵ファイル |

SSH接続では、秘密鍵ファイルの権限が広すぎると安全でないと判断され、接続できない場合がある。

そのため、秘密鍵ファイルは `chmod 400` で安全な権限に設定する。

---

## EC2インスタンスへSSH接続する

### 3. 秘密鍵を指定してSSH接続する

```bash
ssh -i ~/.ssh/udemysample.pem ec2-user@xxx.xxx.xxx.xxx
```

### コマンドの意味

| 部分 | 意味 |
|---|---|
| `ssh` | SSH接続を行うコマンド |
| `-i` | 接続に使う秘密鍵ファイルを指定するオプション |
| `~/.ssh/udemysample.pem` | 使用する秘密鍵ファイル |
| `ec2-user` | EC2へログインするユーザー名 |
| `xxx.xxx.xxx.xxx` | EC2インスタンスのパブリックIPアドレス |

`xxx.xxx.xxx.xxx` の部分は、自分のEC2インスタンスのパブリックIPアドレスに置き換える。

---

## 今日のポイント

- MacではTerminalを使ってEC2へSSH接続できる。
- `.pem` ファイルは秘密鍵なので、安全に管理する必要がある。
- 秘密鍵ファイルは `.ssh` フォルダに移動しておくと管理しやすい。
- SSH接続前に、秘密鍵の権限を `chmod 400` に変更する。
- EC2へ接続するときは、秘密鍵ファイルとパブリックIPアドレスを指定する。

## 自分用メモ

MacでEC2に接続する場合は、以下の流れで作業する。

1. `.pem` ファイルをダウンロードする
2. `.pem` ファイルを `.ssh` フォルダへ移動する
3. `chmod 400` で権限を変更する
4. `ssh -i` を使ってEC2へ接続する

秘密鍵ファイル名とパブリックIPアドレスは、自分の環境に合わせて変更すること。

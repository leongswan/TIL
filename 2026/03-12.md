## 学んだ内容

### 1. AWSの責任共有モデルを理解する
AWSでは、セキュリティとコンプライアンスの責任を  
**AWS側**と**利用者側（お客様）**で分担する。

#### 知識
- AWS責任共有モデル

#### スキル
- AWS責任共有モデルの構成要素を理解する
- お客様の責任を説明できる
- AWSの責任を説明できる
- AWSとお客様が共有する責任を説明できる
- 利用するサービスによって責任範囲が変わることを理解する
  - 例：Amazon RDS
  - 例：AWS Lambda
  - 例：Amazon EC2

#### ポイント
- AWSは**クラウドそのもののセキュリティ**を担当する
- 利用者は**クラウドの中の設定やアクセス管理**を担当する
- マネージドサービスを使うほど、利用者側の管理負担は小さくなる

### 3. AWSアクセス管理機能を特定する

#### 知識

- ルートユーザー保護の重要性
- 最小権限の原則

#### スキル
- 認証情報の管理方法を理解する
  - アクセスキー
  - パスワードポリシー
  - AWS Secrets Manager
  - AWS Systems Manager
- AWSの認証方法を理解する
  - MFA（多要素認証）
  - IAM Identity Center
  - アカウント間のIAMロール

  - グループ
  - ユーザー

  - マネージドポリシー

#### ポイント

## 用語解説

| 用語 | 意味 |
|---|---|
| AWS責任共有モデル | AWSと利用者でセキュリティやコンプライアンスの責任を分担する考え方 |
| ルートユーザー | AWSアカウント作成時に最初に使う特別な管理者アカウント |
| IAM | AWSでユーザー・権限・アクセス管理を行うサービス |
| AWS IAM Identity Center | 複数システムにまたがるユーザー管理やSSOを行いやすくする機能 |
| AWS Marketplace | AWS上で利用できる製品やソリューションが提供されているサービス |

## 概要
今日は、**AWS CLI を使って AWS をコマンドで操作する方法**を学んだ。  
実際の画面では、**S3 のバケット一覧をコマンドで取得**し、  
AWSのWeb画面（GUI）で見える情報をCLIでも確認できることを理解した。

---

## 実際に確認した操作
今回の画面では、**Windows のコマンドプロンプト**から  
S3 バケット一覧を表示する操作が紹介されていた。

### 実行したコマンド
```bash
aws s3 ls
```

### コマンドの意味
- `aws`  
  AWS CLI を実行するコマンド
- `s3`  
  Amazon S3 を対象にする指定
- `ls`  
  一覧表示する命令

つまり、`aws s3 ls` は  
**「S3 バケットの一覧を表示するコマンド」** という意味になる。

---

## 画面から分かったこと
コマンド実行後、画面には以下のような情報が表示されていた。

- S3 バケットの作成日時
- バケット名
- 複数のバケット一覧

また、AWSのS3画面にも同じバケットが表示されていたため、  
**GUIとCLIで同じAWSリソースを確認できる**ことが分かった。

---

## 学んだポイント

### 1. CLIでもAWSリソースを確認できる
AWSはWeb画面だけでなく、**コマンドでも操作可能**。  
同じS3バケットでも、コンソール画面から見る方法と、CLIから確認する方法がある。

### 2. `aws s3 ls` は基本コマンド
S3を学ぶうえで、とても基本的なコマンド。  
まずはこのコマンドで、**現在見えているバケット一覧を確認する**のが出発点になる。

### 3. CLIは繰り返し作業や自動化に向いている
Web画面ではクリック操作が必要だが、CLIならコマンドで素早く実行できる。  
今後は以下のような用途でも役立つ。

- 繰り返し作業
- スクリプト化
- 自動化
- サーバー上でのAWS操作

### 4. 実行には事前設定が必要
AWS CLI を使うには、通常は以下の準備が必要になる。

- AWS CLI のインストール
- アクセスキーの設定
- リージョン設定
- 認証情報の設定（`aws configure` など）

今回はコマンド実行例の確認が中心だったが、  
CLIは**設定が済んでいればすぐにAWS操作ができる**。

### 5. 環境によって警告が出ることがある
画面上では、文字コードに関する警告らしき表示も見えた。  
ただし、その後にバケット一覧は表示されていたため、  
**環境依存の警告が出ても結果自体は取得できる場合がある**ことも分かった。

## GUI操作との違い

| 項目 | AWSコンソール | AWS CLI |
|---|---|---|
| 操作方法 | 画面をクリックして操作 | コマンド入力で操作 |
| 分かりやすさ | 初学者に分かりやすい | コマンド理解が必要 |
| 速度 | 画面遷移が必要 | 慣れると速い |
| 自動化 | しにくい | しやすい |
| 向いている場面 | 初期学習・確認 | 運用・自動化・大量操作 |

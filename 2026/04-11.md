## 1. ユーザーデータの利用
EC2では **ユーザーデータ（User Data）** を使って、インスタンス起動時に自動実行されるスクリプトを設定できる。

### ポイント
- EC2起動時の詳細設定を自動化できる
- Bashスクリプトなどを記述できる
- 起動時に必要な初期設定をまとめて実行できる
- この仕組みは **ブートストラップ** と呼ばれる

### 学んだこと
毎回手作業で設定しなくても、ユーザーデータを使えば  
**サーバー初期化の自動化** ができる。

---

## 2. ストレージの選択
EC2では、直接利用するストレージを追加・設定できる。  
ストレージには大きく分けて、**インスタンスストア** と **EBS** がある。

---

## 3. インスタンスストアとEBS
### インスタンスストア
ホストコンピュータに内蔵されたディスクで、EC2と不可分な物理ストレージ。

#### 特徴
- EC2の一時的なデータ保持に使われる
- EC2の停止・終了とともにデータは消える
- 一時領域として使うのに向いている

### EBS（Elastic Block Store）
ネットワーク接続されたブロックレベルストレージで、EC2とは独立して管理される。

#### 特徴
- EC2を停止・終了してもデータを保持できる
- バックアップとしてスナップショットをS3に保存できる
- 別途EBS料金が発生する
- 永続的なディスクとして使える

### 学んだこと
- **消えてもよい一時データ** はインスタンスストア向き
- **残したいデータ** はEBS向き

---

## 4. ストレージのオプション
EC2ではブロックストレージ以外に、ファイルシステム型のストレージも使える。

### Amazon EFS
- NASに似たファイルストレージ
- ファイルシステムとして利用できる
- 複数のEC2インスタンスから共有アクセスできる

### Amazon FSx for Windows File Server
- Windows File Server と互換性のあるストレージ
- Windows環境向け
- Windows AD やOS、ソフトウェアとの連携が豊富

### 学んだこと
EC2のストレージはEBSだけでなく、  
**共有ファイルシステム用途にはEFSやFSxも選べる**。

---

## 5. タグ設定
タグは、AWSリソースに名前や分類情報を付ける仕組み。

### ポイント
- キー + 値 のセットで設定する
- 例: `Name = GroupA`
- EC2などのリソースのグループ分けに使える
- 権限分けや管理のしやすさにも役立つ
- リソースタイプを指定して、どのリソースに付けたタグか分類できる

### 学んだこと
タグは単なる名前付けではなく、  
**管理・運用・権限制御をしやすくするための重要な仕組み**。

---

## 6. セキュリティグループ
セキュリティグループは、インスタンスへの通信可否を設定する **ファイアーウォール機能**。

### 例
- HTTPアクセスは拒否
- SSHアクセス（ポート22）は許可

### ポイント
- インバウンド通信の許可/拒否を制御する
- 必要な通信だけを開けるのが基本
- EC2の安全な公開に必須

### 学んだこと
EC2を作るだけではなく、  
**どの通信を通すかを厳密に決めることが重要**。

---

## 7. キーペアの利用
EC2インスタンスへ安全にアクセスするために **キーペア** を利用する。

### 仕組み
- 利用者が **秘密鍵** を持つ
- EC2側には対応する **公開鍵** が設定される
- 秘密鍵を使ってインスタンスへアクセスする

### ポイント
- ダウンロードした秘密鍵を大切に保管する必要がある
- 公開鍵と秘密鍵が対応していないとアクセスできない
- VPCでは最大5000個まで作成可能

### 学んだこと
パスワードではなく、  
**公開鍵暗号方式でEC2に安全に接続する** という考え方が重要。

---

## 8. インスタンスの購入方式
EC2には用途に応じて複数の購入方式がある。

### オンデマンドインスタンス
- 通常の購入方式
- 長期契約なし
- 秒単位または分単位で支払う
- 起動、停止、休止、削除、再起動などを自由に制御できる

### リザーブドインスタンス（RI）
- 1年または3年利用を予約することで割引
- オンデマンドより大幅に安くなることがある
- 特定のAZやリージョンのキャパシティ予約タイプもある

### スケジュールドリザーブドインスタンス
- 決まった曜日・時間帯だけ予約する方式
- **2021年に利用停止**

### スポットインスタンス
- AWSの未使用EC2を安価に利用できる
- オンデマンドより低価格
- 中断される可能性がある
- 実行時間に柔軟性がある処理向け

### 学んだこと
EC2は同じ性能でも購入方式でコストが大きく変わるため、  
**使い方に応じて最適な購入方式を選ぶことが大事**。

---

## 9. 物理対応可能なインスタンス
通常の仮想サーバー型EC2以外に、物理サーバーに近い利用方法ができるインスタンスもある。

### Dedicated Instance
- 専用ハードウェアのVPC内で実行されるEC2
- 他のAWSアカウントとは物理的に分離される
- ただし同一AWSアカウント内では共有の可能性がある

### Dedicated Host
- 物理ホストサーバーを1ユーザーが専有する
- 他ユーザーと共有しない
- 物理サーバーに紐づくソフトウェアライセンス利用に向く

### Bare Metal
- 基盤サーバーのプロセッサやメモリに直接アクセス可能
- OSが下層ハードウェアに直接近い形で動作する
- AWSサービスと連携しながら物理サーバーに近い使い方ができる

### 学んだこと
EC2は単なる仮想サーバーだけではなく、  
**専有ホストや物理サーバーに近い構成まで選べる**。

## 今日の学び
- ユーザーデータを使うと起動時の設定を自動化できる
- EC2ストレージには一時的なインスタンスストアと永続的なEBSがある
- EFSやFSxを使えば共有ファイルシステムも構成できる
- タグは管理や権限制御に役立つ
- セキュリティグループで必要な通信だけを許可する
- キーペアで安全にインスタンスへ接続する
- 購入方式によってEC2コストを最適化できる
- Dedicated Host や Bare Metal など物理寄りの利用形態もある

## 一言まとめ
EC2は単なる仮想サーバーではなく、  
**起動自動化、ストレージ設計、通信制御、認証、料金最適化まで含めて柔軟に構成できるサービス** だと学んだ。
